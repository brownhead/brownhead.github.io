Frameworks and Libraries

Boost

Boost is a great library I use whenever I am writing in C++ and am not restricted from using it. Though I certainly haven't used even a majority of libraries in Boost, I have read through all of their summaries and usually have a good idea when it is appropriate to use them.

Flask

I have tried a number of Python web frameworks including Pyramid, web.py, and CherryPy, but none of them felt like they embraced the ideals of Python better than the Flask web framework. I have used it in several of my projects since I began playing with it a year ago.

jQuery

I was a late-comer to the jQuery train and only began using it about four years ago. Now I almost never write JavaScript code without using jQuery.

Windows Presentation Framework (WPF)

My previous exploits into native GUI development were limited to Visual Basic 6 and Qt, so WPF truly wowed me. It is still undoubtedly my favorite tool for creating GUI's.

Miscellaneous

MongoDB

My project Galah uses MongoDB so I have gotten a lot of experience dealing with it. Fortunately, minus some difficulties getting backups to work correctly, MongoDB has performed perfectly and never have we had a problem with it.

nginx

nginx is certainly my favorite web server at the moment. It is incredibly fast, the configuration is many times simpler than Apache, and it has a lot of interest. I have been working with it for nearly two years now.

uWSGI

uWSGI is an interesting application that sits between a web server (such as nginx) and your web application that supports the WSGI interface (yes talking about the two of them in the same conversation gets difficult). I have used it as long as I have been using nginx.
